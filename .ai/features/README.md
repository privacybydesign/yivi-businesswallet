# Features

Durable docs explaining what a capability is and why it is built the way it is.
Committed and cross-tool. Loaded when working on that capability.

Different from conventions: a convention is a rule to follow while editing
("org-scoped routes compose RequireUser then Authorize"); a feature doc explains
a capability's design and rationale ("auth is Yivi via a standalone daemon
because ..."). One constrains, one explains. Keep them apart.

Written by the Harvest step in .ai/plans/README.md, not hand-maintained ad hoc.
